به اولین پست وبلاگ من خوش آمدید! در این پست قصد دارم تجربه ساخت این وب‌سایت با استفاده از GitHub Pages را با شما به اشتراک بگذارم.

GitHub Pages چیست؟

GitHub Pages یک سرویس رایگان برای میزبانی سایت‌های استاتیک است که مستقیماً از مخزن (repository) گیت‌هاب شما ساخته می‌شود. این سرویس برای سایت‌های شخصی، پروژه‌ها و مستندات بسیار مناسب است.

چرا از GitHub Pages استفاده کنیم؟

استفاده از GitHub Pages چندین مزیت دارد:

  • رایگان است
  • به راحتی با گیت‌هاب یکپارچه می‌شود
  • بسیار امن است
  • برای سایت‌های استاتیک عملکرد بسیار سریعی دارد
  • همیشه در دسترس است (آپ‌تایم بالا)

نحوه استفاده از GitHub Pages

برای استفاده از GitHub Pages، ابتدا باید یک مخزن در گیت‌هاب ایجاد کنید. نام این مخزن باید به صورت username.github.io باشد، که username همان نام کاربری شما در گیت‌هاب است.

نکته: اگر می‌خواهید برای یک پروژه خاص سایت بسازید، می‌توانید یک شاخه به نام gh-pages ایجاد کنید یا تنظیمات GitHub Pages را در بخش تنظیمات مخزن خود تغییر دهید.

گام‌های اساسی

  1. ایجاد یک مخزن جدید در گیت‌هاب
  2. کلون کردن مخزن در سیستم خود
  3. افزودن فایل‌های HTML, CSS و JavaScript
  4. کامیت و پوش کردن تغییرات به گیت‌هاب
  5. فعال کردن GitHub Pages از تنظیمات مخزن

نتیجه‌گیری

استفاده از GitHub Pages یک راه عالی برای ایجاد وب‌سایت‌های استاتیک است. من تصمیم گرفتم وبلاگ خود را با این روش راه‌اندازی کنم و تا اینجا بسیار راضی هستم. در پست‌های بعدی، درباره موضوعات مختلف دیگر خواهم نوشت.

برای دریافت آخرین اخبار و مقالات، حتماً این وبلاگ را دنبال کنید!
